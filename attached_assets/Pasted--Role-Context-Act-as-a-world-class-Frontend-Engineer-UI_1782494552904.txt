# Role & Context
Act as a world-class Frontend Engineer, UI/UX Graphics Designer, and Web Physics/Animation Expert. You are updating the desktop navigation bar component for the THORX premium web application.

Current State: Desktop navbar features rectangular buttons containing a layout of `[icon, text]`. Currently, an `.active` state modifies the background color to orange and adds a translational "pop out" offset.

# Objective
Refactor the active state completely. Implement a highly interactive, luxury-grade, physics-driven Canvas or SVG-Verlet animation for the selected navigation item's icon to match "THORX million-dollar standards."

# Technical & Implementation Requirements

### 1. Visual Refactoring (De-activating the Old State)
- Remove all legacy orange backgrounds, scale offsets, and pop-out visual changes on active buttons.
- The active nav button container must remain visually identical to unselected buttons in the DOM layout.

### 2. Architectural Setup (The Physics Canvas)
- Overlay a transparent HTML5 `<canvas>` (or a high-performance interactive `<svg>`) absolute-positioned across the header/navbar bounds to handle the physics rendering. 
- Ensure it handles window resizing dynamically with proper pixel-ratio scaling (`devicePixelRatio`) to prevent any blurriness.

### 3. "Drop & Hang" Pendulum Physics Engine
When a navigation item becomes active:
- **Detachment:** Hide the static inline icon and instantly spawn a dynamic vector version at its exact coordinates on the physics canvas.
- **The Constrained Rope:** Attach the icon to its anchor point (the center-bottom of the navbar item) via a simulated elastic rope/spring.
- **Verlet Integration / Hooke's Law:** Program a physics loop implementing gravity ($g$), pendulum angular acceleration, air resistance/damping (to prevent infinite swinging), and spring tension ($k$) using Hooke's Law:
  $$F = -k \cdot x$$
- **Initial Velocity:** On click, give the icon a slight randomized angular or downward velocity so it falls, bounces elastically at the rope's maximum length, and swings like a realistic pendulum before settling.

### 4. Direct Pointer Interaction (Grab, Drag, & Snap)
- **Pointer Events:** Implement `pointerdown`, `pointermove`, and `pointerup` listeners to support both mouse and touch interaction on the canvas icon element.
- **Cursor Tracking:** While dragging, the icon center must track the cursor position with a slight spring-mass inertia delay (do not rigidly glue it to the pointer; give it weight).
- **Rope Stretching & Tension:** If the user drags the icon past the rope's rest length, calculate and render a realistic elastic narrowing/stretching visual for the rope line. Increase the restoring spring force proportionally.
- **Release (Spring-back):** Upon `pointerup`, release the icon back to the physics simulation, letting it violently snap back, oscillate, and naturally settle back into its hanging resting state via momentum and friction damping.

### 5. Code Quality & Performance
- The entire animation loop must run inside a `requestAnimationFrame` block.
- Optimize calculations to ensure a locked 60 FPS / 120 FPS on premium displays with zero layout thrashing.
- Keep the physics code self-contained, modular, and cleanly documented so it can be easily adjusted (e.g., exposing variables for `gravity`, `friction`, `elasticity`).
