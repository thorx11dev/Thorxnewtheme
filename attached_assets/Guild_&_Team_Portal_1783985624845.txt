Guild & Vault Admin/Team Portal
What & Why
Give the Team/Admin Portal full visibility and control over the new Guild/Escrow Vault/Points Ledger system from thorx_master_plan.md: configuration sliders for vault %, multipliers, and conversion rate, a Guild moderation panel (strikes/freeze/disband), and ledger/vault visibility inside the existing admin task manager and settings screens. This depends on the guild/vault/points-ledger backend and APIs already existing.

Done looks like
SystemSettingsManager.tsx gains a new configuration section (matching its existing card style) with controls for: escrow vault hold percentage, weekly goal targets per guild rank, release multipliers per guild rank, and the live points-to-PKR conversion rate — all reading/writing the new systemConfig keys via the existing bulk-config GET/PATCH endpoints.
A new "Guilds" section appears in AdminSidebar.tsx/AdminLayout.tsx navigation, showing a searchable/filterable list of all guilds with rank, member count, vault balance, strikes, and status (active/frozen), with per-guild actions to add/clear a strike, freeze/unfreeze payouts, or disband — reusing the existing admin card/table styling used by UserManager.tsx/RiskWatchlistPanel.tsx.
The withdrawal fee split shown in PayoutControl.tsx (or wherever admin reviews withdrawals) reflects the simplified single-level referral share instead of the old L1+L2 breakdown.
TaskManager.tsx (internal task manager) can surface or link guild-related weekly-goal / vault-release events if that's a natural fit for admin ops visibility, without duplicating the new Guilds panel's core functionality.
All new admin views enforce the same role/permission checks already used by existing admin panels.
Out of scope
Any backend/schema/API work — this task only consumes existing or to-be-existing guild/vault/ledger admin endpoints; if an endpoint you need doesn't exist yet, flag it rather than inventing new backend logic here.
User Portal changes (separate task).
Changing the existing risk-scoring engine's weighting UI already present in SystemSettingsManager.tsx.
Steps
Vault & ledger config controls — Add sliders/inputs for VAULT_HOLD_PCT, per-rank weekly targets, per-rank release multipliers, and CONVERSION_RATE/REFERRAL_FEE_SHARE_PCT to SystemSettingsManager.tsx, following the existing EconomicControl/save-mutation pattern in that file.
Guild moderation panel — Build a new admin component/view listing all guilds with drill-down into members, vault history, and weekly cycle status, plus strike/freeze/unfreeze/disband actions, and wire it into the sidebar navigation.
Withdrawal/payout view update — Update wherever admin currently sees the L1/L2 commission breakdown for a withdrawal to show the single Level-1 referral share instead.
Task manager integration (optional/light-touch) — If natural, surface upcoming/failed weekly guild resolutions as visibility items in TaskManager.tsx without rebuilding guild moderation there.
