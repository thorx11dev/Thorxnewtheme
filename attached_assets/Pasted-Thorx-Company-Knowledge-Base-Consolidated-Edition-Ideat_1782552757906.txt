Thorx — Company Knowledge Base
Consolidated Edition (Ideation & Planning + Technical Architecture) Target Market: Pakistan (First Phase) Primary Integration: JazzCash, EasyPaisa


1. Foundational Idea
1.1 The Idea
Thorx exists to change the trend of advertising content and lead generation — and, moreover, to help people earn money online.
1.2 The Loophole (The Problem)
Most advertisers run their video ads on social media platforms such as YouTube, Instagram, Facebook, and TikTok. However, these ads pull less attention from viewers, which means fewer leads for the advertiser.
1.3 The Thorx Solution
This is the gap Thorx fills. On Thorx, users don't just passively pay attention to ads — they also help advertisers and digital workers generate real leads, with the help of Thorx's own LeadX system.
1.4 Core Value Proposition (Expanded)
Thorx is a digital platform engineered to disrupt online advertising by turning human attention into currency. Traditional digital advertising suffers from budget leakage caused by passive, ignored, or bot-driven impressions. Thorx addresses this with a dual-phase verification layer that bridges passive viewing and active engagement — ensuring better conversion rates for advertisers while rewarding users for verified attention.


2. Actors
Three actors operate within the Thorx ecosystem:

Earners — users who pay attention to ads/offers in exchange for money.
Lead-Seekers — advertisers seeking leads for their ads or offers.
THORX (the platform itself) — the actor responsible for ensuring everything works well between the two sides.

(In the technical architecture, Earners are referred to as "Users" and Lead-Seekers as "Advertisers.")


3. Core Action Flows
Earner flow:

Lead-Seeker flow:

Go to THORX

Launch an ad or an offer

THORX's role:

Ensure everything works well for both sides


4. What Thorx Offers
Thorx offers two portals, each with its own features and systems:
4.1 User Portal (Earners)
Covers the Earners' area. Helps Earners link with advertisers' ads and offers, pay attention to them, and earn money through local bank/payment options.
4.2 Advertiser Portal (Lead-Seekers)
Covers the area for lead seekers. Helps them use Thorx's pre-defined system called LeadX. Lead-seekers are not locked in — they can also choose to ignore the LeadX system and go directly to launching their own ads or offers.

6. Platform Architecture: The Three Engines
Engine A: The Attention Marketplace (Core USP)
Engine A is designed to maximize ad memorability and conversion through a structured, two-part user flow.

Phase 1 (The Video Player): The user views a 15–25 second video ad, delivered via a proprietary, pre-built waterfall video player.
The Waterfall Model: If Ad Network 1 fails to load or runs out of inventory, the player automatically drops to Network 2, then Network 3, ensuring zero downtime and maximum fill rates.
Targeting: Upon account creation, users input their specific interest profiles. The waterfall system filters and serves relevant ads to optimize natural conversion rates.
Phase 2 (The Sandbox Web Panel): Once the ad finishes, the user requests to move to the next step. The advertiser's product landing page loads seamlessly inside a dedicated Thorx sandbox container (<iframe> or secure sub-window).
The Super AI Attention Detector (Hidden): The user must explore the page for 15 seconds. To prevent users from gaming the system, the tracking mechanics are completely invisible; the user only sees a premium, minimalist countdown timer. The hidden script runs in the parent frame, compiling a behavioral payload measuring:
Tab Visibility API: Instantly pauses the countdown if the user switches browser tabs or minimizes the window.
Micro-Movement Delta: Detects cursor coordinates (x, y) or touch inputs to ensure the device has not been left flat on a desk.
Scroll Vector: Tracks if the user scrolls down at least 10–20% of the viewport height, proving active reading/exploration.
Payout Condition: If real attention is verified, the network updates the user's balance. If behavioral thresholds aren't met, no payout is issued.
Engine B: AI-Driven CPA Offers
Engine B serves as a high-yield secondary revenue stream featuring actions like downloading apps, leaving social media reviews, or completing registrations.

Curation: The Thorx team cherry-picks the most reliable, highest-converting offers from established Cost-Per-Action (CPA) networks.
Automated Proof Vetting: Users upload image/video proof of completed tasks. Instead of manual bottlenecks, a custom AI Agent processes submissions through a multi-tier fraud check:
Metadata/Hash Extraction: Instantly drops duplicate images, recycled proofs, or files with mismatched creation timestamps.
Advanced OCR + LLM Analysis: Extracts text from the screenshot to verify target handles, specific comment text, and modern UI structures.
Escrow Release: Approved tasks enter a "Pending" escrow status before being cleared by the admin panel for final wallet release.
Engine C: The 3-Division Referral Matrix
Engine C drives organic viral growth via a multi-layered referral infrastructure spanning three tiers (User A → User B → User C). Commission structures are tied to payout requests, turning user acquisition into a self-sustaining ecosystem.

Tier 1 Referral (Level 1): When User B initiates a withdrawal, a 15% referral commission is deducted from their gross earnings and distributed to User A.
Tier 2 Referral (Level 2): When User C requests a payout, User B receives a 15% commission, and User A receives a 7.5% commission from User C's gross earnings.
The Solo Loop / Admin Charge: Any user who joins the platform organically, without a referral link or code, is subject to a flat 15% Admin Charge upon withdrawal.


7. Financial Mechanics & Wallet Logic
7.1 The Net-First UI Strategy
To combat user churn and the psychological friction associated with steep payout deductions, Thorx implements a strict Net-First UI Pattern:

Users never see an inflated gross balance that gets heavily slashed at checkout.
All available wallet views, task rewards, and dashboards calculate network fees silently in the background, displaying exactly what the user can pull out: "What You See Is What You Get."

8. Technical Guardrails & Deployment Strategy
Platform Medium: Responsive Web First (optimized for both mobile web viewports and desktop browsers). This cuts out app store approval delays and allows for immediate hot-fixes to the hidden tracking scripts.
Ad-Blocker Strategy — The Gateway Block: A custom script executes immediately upon web entry. If it catches an active ad-blocker or ad-blocking browser (like Brave), it serves a premium modal blocking access to earning loops until Thorx is whitelisted.
Localization Layer: Tailored completely for Pakistan's digital landscape. Payouts hook directly into JazzCash and EasyPaisa workflows.
