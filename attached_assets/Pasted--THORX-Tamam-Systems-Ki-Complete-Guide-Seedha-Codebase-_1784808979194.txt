## 1️⃣ TX-POINT SYSTEM
**"Asli paisa points mein badalta hai"**

THORX mein user ko primary view mein seedha PKR (rupees) nahi milta — pehle **TX-Points** milte hain.

### Kaise Kaam Karta Hai:
* Jab user koi earning karta hai, system us ki PKR value ko ek rate se multiply karta hai.
* Yeh rate `CONVERSION_RATE` kehlata hai — default value hai **100**.
* **Formula:**  
  $$	ext{User PKR Share} 	imes 	ext{CONVERSION_RATE} = 	ext{TX-Points Credited}$$  
  *(Matlab: Rs. 1 PKR = 100 TX-Points)*

### Thorx Card Draw (Variance System):
* Earning seedhi fixed nahi hoti — ek "card draw" hoti hai jis mein **±10% variance** hota hai.
* *Example:* Agar aapka hissa Rs. 1 (100 TX-Points) banta hai, toh 90 se 110 TX-Points ke beech kuch bhi mil sakta hai.
* **A-Rank** users ko **+5% extra**, **S-Rank** ko **+10% extra** variance potential milta hai.
* Yeh variance `ILLUSION_VARIANCE_PCT` system config se control hota hai.

### Database Storage:
* `users.txPointsBalance` mein points store hote hain.
* Real PKR estimation withdrawal se pehle ledger history scan se calculate hoti hai.

## 2️⃣ MONEY FLOW & LEDGER-BASED WITHDRAWAL SYSTEM
**"Paise kaise chalte hain — earning se account tak"**

### Step 1 — Earning & Ledger Snapshot
* User ad dekhta hai ya task karta hai → Exact PKR value (`earnedAmountPkr`) calculate hoti hai.
* Thorx Card Draw lagti hai → TX-Points `txPointsBalance` mein credit hote hain.
* **Crucial:** Har transaction ke waqt precise real PKR value aur conversion snapshot `earn_events` / ledger mein permanently log ki jaati hai.

### Step 2 — Thorx Ka Hissa (Engine Cut)
Har engine mein Thorx apna share pehle rakhta hai:
* **Engine A (Ads):** Thorx 40%, User 60%
* **Engine B (CPA Tasks):** Thorx 40%, User 60%
* **Engine C (Guild):** Thorx 20%, Guild Pool 35%, User 45%
* **Engine Indirect:** Sirf PS milta hai, koi points/PKR nahi.

### Step 3 — Ledger-Based Exact Withdrawal (Points → Real Money)
Withdrawal ke waqt system average global rate se points ko divide karne ki bajaye **Ledger-Based Historical Audit** karta hai:

1. **Ledger Scan:** System user ke unwithdrawn points ke mutalliq tamam historical `earn_events` database log scan karta hai.
2. **Exact PKR Sum:** Har task/ad view ke waqt jo real PKR value (`earnedAmountPkr`) calculate hui thi, un sab ka exact sum lagaya jata hai.
3. **Breakdown Calculation (`calculateWithdrawalBreakdown`):**
   * **Gross PKR:** $\sum (	ext{Historical Earned PKR Values for unwithdrawn points})$
   * **Platform Fee:** $	ext{Gross PKR} 	imes 	ext{WITHDRAWAL_FEE_PCT}$ *(Default 15%)*
   * **Referral Commission:** $	ext{Platform Fee} 	imes 	ext{REFERRAL_FEE_SHARE_PCT}$ *(Default 50%)*
   * **Thorx Net:** $	ext{Platform Fee} - 	ext{Referral Commission}$
   * **User Receives:** $	ext{Gross PKR} - 	ext{Platform Fee}$

> **Why Ledger-Based?**  
> Agar future mein conversion rate badle, tab bhi user ko us ke har task ki exact waqt ki real PKR earning mile gi — na user ka nuqsan hoga, na Thorx ko financial loss.

## 3️⃣ REFERRAL SYSTEM
**"Aik ka kaam, doosre ko bhi faida"**

* Har user ka unique `referralCode` hota hai.
* Referrer ko commission **tab milta hai jab referred user withdrawal karta hai** (Withdrawal fee ka 50%).
* **Points-Only Mandate:** User Portal UI mein referral earnings strictly **TX-Points** ke format mein mask/display hoti hain taake app ka format consistent rahe. Final withdrawal screen par hi exact estimated PKR convert ho kar dikhta hai.
* Strictly **1-Tier Referral** system (no multi-level chain).
* All transactions `referral_commissions` table mein record hote hain.
