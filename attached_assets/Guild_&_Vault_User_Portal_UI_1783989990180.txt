Guild & Vault User Portal UI
What & Why
Bring the Thorx Master Plan's user-facing mechanics to life in the User Portal: the Scratch Card earning illusion, a Guild dashboard (create/join, vault status, weekly goal, strikes), and a points-to-PKR ledger/conversion view, replacing the current plain "PKR credited" toast flow with the new points-first experience described in thorx_master_plan.md. This depends on the guild/vault/points-ledger backend and APIs already existing.

Done looks like
Completing an ad or task shows an animated Thorx Scratch Card revealing a randomized point breakdown (e.g. "70 Base + 20 Guild Bonus") instead of the current plain success toast, sourced from the real points_ledger entry the backend created for that action.
A new "Guild" section is reachable from the User Portal's main navigation (alongside Dashboard/Work/Referrals/Payout/Help), where a user with no guild can create or join one, and a user with a guild sees: guild name/rank, members list with captain indicator, current vault balance, weekly goal progress bar, strike count/status, and countdown to weekly resolution.
A ledger/wallet view (e.g. within Dashboard or Payout) shows the user's TX-Points balance alongside its real-time locked PKR equivalent, with a scrollable history of points_ledger entries (source type, points, locked PKR value, date) — the existing withdrawal flow continues to work against the real PKR balance without behavior changes.
Referral view continues to work but reflects the simplified 1-level model (no Level-2 mentions) if it currently surfaces multi-level information.
All new views are visually consistent with the existing User Portal design system (same rounded/bordered card style, typography, and color usage already used in client/src/pages/UserPortal.tsx), and work responsively on mobile since this is a mobile-first bottom-nav app.
Out of scope
Any backend/schema/API work — this task only consumes existing or to-be-existing guild/vault/ledger endpoints; if an endpoint you need doesn't exist yet, stub against the documented shape and flag it rather than inventing new backend logic here.
Team/Admin portal changes (separate task).
Changing the existing five-tier named rank badges/colors — Guild rank (E–S) is a new, separate visual element, not a replacement.
Steps
Scratch Card component — Build a reusable scratch/reveal animation component that displays the point breakdown returned by the ad-view/task-completion API response, triggered right after a successful earn action in place of (or layered on top of) the current toast.
Guild Dashboard view — Add a new nav entry and view for guild creation/joining (search or invite-code based), and for members, a status view: vault balance, weekly target progress, member list, strikes, and rank.
Ledger / Conversion view — Add a wallet sub-view showing TX-Points balance, its locked PKR equivalent, and a paginated ledger history list, wired to the new ledger history endpoint.
Referral view cleanup — Update the referrals section's copy/data to reflect single-level referral sharing only.
State & data wiring — Use the existing React Query (apiRequest) patterns already used throughout UserPortal.tsx for all new data fetching/mutations, keeping consistent loading/error/toast handling with the rest of the file.
