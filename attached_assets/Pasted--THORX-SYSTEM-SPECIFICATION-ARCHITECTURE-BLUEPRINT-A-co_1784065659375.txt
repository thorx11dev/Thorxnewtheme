🚀 THORX SYSTEM SPECIFICATION & ARCHITECTURE BLUEPRINT
A comprehensive system design document for the development team and Replit Agent.
SECTION 1: SYSTEM OVERVIEW & COMPLIANCE
To establish 100% legal compliance within Pakistan and remove any resemblance to multi-level marketing (MLM) schemes, Thorx has transitioned to a clean, direct-value network:
 * No Multi-Level Referrals: All multi-level marketing and secondary/tertiary referral levels are permanently deleted.
 * Direct 1-Tier Referral System: The system only rewards the direct inviter (User A) from the platform's processing fee during User B's withdrawal.
 * Flat Platform Fee: A single, clean, public-facing 15\% platform fee is charged on withdrawals, replacing all complex admin fee structures.
SECTION 2: THE 3-WAY EARNING REVENUE MODEL (THORX PROFIT)
Thorx sustains its operations and generates direct and indirect revenue through three pathways:
[User Task Completion]
       │
       ├─► Solo Tasks (Engine A/B) ──► 40% Direct Thorx Profit
       │
       ├─► Guild Tasks (Engine C)  ──► 20% Direct Thorx Profit + 35% Guild Weekly Pool
       │
       └─► Indirect Tasks ─────────► 100% Thorx Profit (Subs, Video Views, Codes)

 * Withdrawal Fee Share: Thorx collects a 15\% fee during user withdrawals. A percentage (Ref\%, configured by Admin) of this fee goes to the direct referrer, while the remaining portion is pure Thorx profit.
 * Indirect Action Tasks: Integrated into Guild tasks (e.g., subscribing to Thorx YouTube, watching videos for hidden codes, following Thorx TikTok). The traffic and engagement generate indirect organic revenue.
 * Revenue Splitting (Ads & CPA Offers):
   * Engine A (Ads) & Engine B (CPA Offers) [Solo Tasks]:
     * 40\% is automatically retained as Thorx Platform Profit.
     * 60\% is allocated as User Payout (converted to points).
   * Engine C [Guild Tasks]:
     * 20\% is retained as Thorx Direct Profit.
     * 20\% goes to the Guild Weekly Bonus Pool.
     * 15\% goes to the Guild Vault Pool (Combined under-the-hood with the Bonus pool as a 35\% "Guild Weekly Bonus Pool").
     * 45\% is paid out immediately to the active Guild Member as TX-Points.
SECTION 3: THREE USER ROLES & CUSTOM PORTALS
Users are classified into three distinct categories, each experiencing a customized interface:
1. Simple User
 * Cannot access Engine C (Guild Panel/Guild Tasks).
 * Can only view and complete Engine A (Video/Ad Tasks) and Engine B (CPA Offers).
 * Default Engine C screen points to the Public Guild Discovery Area.
2. Guild Member
 * Belongs to a Guild (limited to 10 members max per guild).
 * Operates under a Guild Captain.
 * Default Engine C screen is the Guild Member Panel (not the Public Area).
 * Accesses exclusive Engine C Guild Tasks.
3. Guild Captain
 * The creator of the Guild (by default) or assigned by the previous captain.
 * Can transfer leadership to any Guild Member.
 * Subject to replacement by Thorx Admin if inactive or reported by members.
 * Manages the Guild roster using the Captain Portal.
SECTION 4: THE GUILD SYSTEM & SOCIAL LANDSCAPE
The Guild System is gamified to resemble a job-hunting and business-management environment.
A. Public Discovery Area (For Simple Users & Guests)
 * Social & Gamified Leaderboard: Displays Guilds ranked from top to bottom based on their Guild Performance Score (GPS).
 * Guild Transparency Info: Displays Guild name, description, avatar, total TX-Points earned, current rank, and available slots (out of 10).
 * The Job Application System:
   * If a vacancy exists and the user meets the Minimum Rank requirement set by the Captain, they can submit an application.
   * Application Letter: The user must write a personalized message (cover letter/proposal) pitching why they are a valuable asset to the team.
B. Guild Member Panel (The Worker's Hub)
 * Weekly Target Progress Bar: A dynamic indicator showing current points vs. the weekly milestone target set by Admin.
 * Contribution Board: Displays the member's personal weekly contribution vs. an internal Top 5 members leaderboard.
 * Team Communication Suite:
   * Guild Chat: A group chat interface (similar to WhatsApp).
   * Direct Captain Channel: A dedicated 1-on-1 text channel between the member and the Captain.
 * Engine C Guild Tasks: Exclusive high-payout CPA offers and promotional tasks (YouTube/TikTok subscription tasks) managed by the admin.
C. Captain Portal (The Management Suite)
 * Roster Controller & Kick Button: Displays usernames and weekly contributions of all 10 members, with an option to remove/kick underperforming members.
 * Application Review Board: Displays applicant details, cover letters, their Performance Score (PS), and their platform history (number of successful ads/offers completed vs. failed/fraudulent attempts).
   * Accept / Reject Actions: Captain can approve entry or reject with a required Why Reject text statement.
 * The Nudge System: A button next to inactive members (zero contribution) that triggers a direct mobile push notification: "Yar, aap ke Captain ne yaad kiya hai! Jaldi se team tasks complete karo..."
 * Weekly Goal Metrics: Displays statistical charts showing successful vs. failed weekly targets over past weeks.
 * MVP Selection: Once a week, the Captain can designate one outstanding member as the "MVP of the Week", pinning a special badge to their profile.

SECTION 7: PROPOSED DATABASE SCHEMA FOR REPLIT AGENT
-- Enums for Ranks
CREATE TYPE rank_tier AS ENUM ('E-Rank', 'D-Rank', 'C-Rank', 'B-Rank', 'A-Rank', 'S-Rank');

-- Users Table Updates
ALTER TABLE users 
ADD COLUMN role VARCHAR(20) DEFAULT 'simple', -- 'simple', 'member', 'captain'
ADD COLUMN performance_score INT DEFAULT 0,
ADD COLUMN rank rank_tier DEFAULT 'E-Rank',
ADD COLUMN balance_cash_pkr DECIMAL(10,2) DEFAULT 0.00, -- For Referral earnings cashouts
ADD COLUMN last_active_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP;

-- Guilds Table
CREATE TABLE guilds (
    id SERIAL PRIMARY KEY,
    name VARCHAR(50) UNIQUE NOT NULL,
    description TEXT,
    avatar_url VARCHAR(255),
    captain_id INT REFERENCES users(id),
    guild_performance_score INT DEFAULT 0,
    guild_rank rank_tier DEFAULT 'E-Rank',
    member_capacity INT DEFAULT 10,
    min_rank_required rank_tier DEFAULT 'E-Rank',
    weekly_target_points INT DEFAULT 50000,
    current_weekly_points INT DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Guild Members Mapping
CREATE TABLE guild_members (
    id SERIAL PRIMARY KEY,
    guild_id INT REFERENCES guilds(id) ON DELETE CASCADE,
    user_id INT REFERENCES users(id) ON DELETE CASCADE,
    weekly_points_contributed INT DEFAULT 0,
    joined_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Membership Applications
CREATE TABLE guild_applications (
    id SERIAL PRIMARY KEY,
    guild_id INT REFERENCES guilds(id),
    user_id INT REFERENCES users(id),
    cover_letter TEXT,
    status VARCHAR(20) DEFAULT 'pending', -- 'pending', 'approved', 'rejected'
    rejection_reason TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Financial Ledger (The Bulletproof Ledger)
CREATE TABLE user_transactions (
    id SERIAL PRIMARY KEY,
    user_id INT REFERENCES users(id),
    engine_type VARCHAR(10), -- 'Engine_A', 'Engine_B', 'Engine_C'
    points_credited INT NOT NULL, -- Random output shown on Card
    real_pkr_value DECIMAL(10,4) NOT NULL, -- Backend-calculated actual money
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Referral Commissions Ledger
CREATE TABLE referral_commissions (
    id SERIAL PRIMARY KEY,
    referrer_id INT REFERENCES users(id),
    invitee_id INT REFERENCES users(id),
    withdrawal_id INT, -- Linked once payout processed
    commission_amount_pkr DECIMAL(10,2),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

SECTION 8: ADMIN CONTROL SUITE REQUIREMENTS
 * Profit Split Sliders: UI inputs to change the percentages of Engine Profit splits and the Referral Share Percentage (Ref\%) of the secret fee.
 * Ledger Validator Tool: A reconciliation algorithm that checks the user's total points vs. the sum of their real_pkr_value before payout approval to detect potential exploits.
 * Simulation Sandbox: Testing tool to simulate Thorx Card outcomes based on inputs of CPA network revenue.
 * Guild Moderation: Admin override capabilities to manually adjust Guild targets, reset inactive Captains, or manually remove violating members.
