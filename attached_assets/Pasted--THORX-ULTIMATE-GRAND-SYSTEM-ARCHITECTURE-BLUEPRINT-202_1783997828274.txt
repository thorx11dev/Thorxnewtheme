## SECTION 1: THE NO-REAL-MONEY VISIBILITY RULE (THE POINT ILLUSION)

To maximize gamification and eliminate the psychological barrier of micro-earnings, the platform is strictly a **Points-Only Ecosystem** in normal usage.

### 1.1 The Rules of Points Display
*   **0% Cash Visibility:** Throughout the entire user app, no user shall see Rs, PKR, or real currency labels on standard screens, dashboard cards, history logs, or cards. Everything is displayed as **TX-Points** or **TX-Gold/Points**.
*   **The Conversion Exception:** The ONLY time real money (PKR) becomes visible is when a user enters the **Conversion Room** inside the Payout section to explicitly initiate a withdrawal request. 
*   **Behind-the-Scenes Locking:** The backend still calculates and records the precise `locked_pkr_value` in the ledger database at the exact millisecond of every event, ensuring no financial loss or calculation issues for Thorx when point-to-PKR conversion rates fluctuate globally.

## SECTION 2: TASK SYSTEM REVOLUTION & PAYOUT DE-COUPLING

### 2.1 Transition to Weekly Task System (Guild Members Only)
*   **Deprecated:** The "Daily Tasks" system is completely deprecated and removed.
*   **New Weekly Tasks:** High-reward Weekly Task Panels are introduced.
*   **Access Control:** These Weekly Tasks are **ONLY visible and accessible to active members of a Guild in Engine C**. Solo users (who are not in any Guild) cannot see or participate in these high-value tasks.

### 2.2 Unlocking Payouts (No More Daily Task Blocks)
*   **Legacy Block Removed:** Previously, the payout/withdrawal section was locked if a user did not complete a daily task. **This restriction is completely removed.**
*   **New Rule:** Access to the payout/withdrawal section is always open and available, provided the user meets the minimum point conversion threshold. Completing tasks is no longer a prerequisite for requesting a payout.

## SECTION 5: DATABASE SCHEMA (POSTGRESQL)

```sql
-- Core Users Table
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    username VARCHAR(50) UNIQUE NOT NULL,
    device_id VARCHAR(255) UNIQUE NOT NULL, -- Strict device-lock
    personal_rank VARCHAR(1) DEFAULT 'E', -- S, A, B, C, D, E
    personal_score INT DEFAULT 0,
    tx_points_balance INT DEFAULT 0, -- Points waiting to be converted
    cash_wallet_balance DECIMAL(10,2) DEFAULT 0.00, -- Real PKR after conversion, ready for Payout Section
    referrer_id INT REFERENCES users(id) ON DELETE SET NULL
);

-- Guilds / Engine C Table
CREATE TABLE guilds (
    id SERIAL PRIMARY KEY,
    name VARCHAR(100) UNIQUE NOT NULL,
    description TEXT,
    avatar_url VARCHAR(255) DEFAULT 'default_guild_avatar.png',
    captain_id INT REFERENCES users(id) ON DELETE RESTRICT,
    guild_rank VARCHAR(1) DEFAULT 'E', -- S, A, B, C, D, E
    guild_score INT DEFAULT 0,
    strikes INT DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Membership Table
CREATE TABLE guild_members (
    id SERIAL PRIMARY KEY,
    guild_id INT REFERENCES guilds(id) ON DELETE CASCADE,
    user_id INT REFERENCES users(id) ON DELETE CASCADE,
    joined_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    status VARCHAR(20) DEFAULT 'pending', -- pending, approved
    UNIQUE(user_id) -- User can join only 1 Guild
);

-- Real-time Valuation Ledger
CREATE TABLE points_ledger (
    id SERIAL PRIMARY KEY,
    user_id INT REFERENCES users(id) ON DELETE CASCADE,
    source_type VARCHAR(50) NOT NULL, -- 'ad_view', 'cpa_offer', 'referral', 'vault_release'
    points_displayed INT NOT NULL,     -- The gamified illusion points
    locked_pkr_value DECIMAL(10,4) NOT NULL, -- Exact PKR value locked at that instant
    conversion_rate_used DECIMAL(10,4) NOT NULL,
    is_converted BOOLEAN DEFAULT FALSE, -- Flips to TRUE once converted in Payout Room
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Real-time Engine C Group Chat Table
CREATE TABLE engine_c_messages (
    id SERIAL PRIMARY KEY,
    guild_id INT REFERENCES guilds(id) ON DELETE CASCADE,
    sender_id INT REFERENCES users(id) ON DELETE CASCADE,
    message TEXT NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
